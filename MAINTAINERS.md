# Maintainers

This file defines maintainership and decision-making for **47Project Framework**.

## Roles

### Maintainers
Maintainers can:
- Approve and merge pull requests
- Cut releases and sign artifacts
- Change schemas, core behavior, and security policy (preferably via RFC)

### Reviewers
Reviewers can:
- Review PRs and request changes
- Maintain modules or areas they own

## Ownership
- **Core / Framework/**: Maintainers
- **Schemas / Trust / Policy**: Maintainers (changes require RFC)
- **Modules/**: Module owners + Maintainers
- **Docs**: Anyone (maintainers approve)

## Decisions
- Small changes: PR + 1 maintainer approval.
- Schema/trust/plan-runner changes: RFC + 2 maintainer approvals (or one approval + 72h no objections).
